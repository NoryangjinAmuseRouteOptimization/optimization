# 솔버 베이스라인 & end-to-end 하니스 (P3 임계경로 부트스트랩)

> 도구: `tools/bench_solver.py` — 실제 제출 진입점을 40개 인스턴스에 돌려
> `utils.check_feasibility`로 **실제 대회 목적함수**를 측정(패킹 밀도 아님).

## 배경: 팀 상태 점검 결과

리포지토리에 **P1 커밋만** 존재(스케줄러/제출 하니스 부재). 즉 제출 가능한
end-to-end 파이프라인이 없어, 누구의 개선도 점수로 확인 불가 → 임계경로 차단.
P1이 P3 하니스를 부트스트랩해 이를 해소.

## 발견 1: 제공된 baseline_greedy는 제출 불가 수준

`--solver baseline` 측정(전용 CPU, jobs=1):

| 인스턴스 | 시간제한 | 실제 소요 | feasible |
|---|---:|---:|---|
| prob_21 | 15s | **47.6s** | ❌ stage 3 |
| prob_22 | 15s | **21.1s** | ❌ stage 2 |
| prob_23 | 15s | **102.3s** | ❌ stage 2 |
| prob_24 | 15s | **80.8s** | ❌ stage 2 |

❌ **시간제한을 3~7배 초과 + infeasible.** 서버 기준 각 **−1점**(시간초과 AND 비실현).
원인: 보수(repair) 루프의 시간가드가 패스 사이에서만 검사 → 한 패스 내
`check_feasibility`(대형 인스턴스 O(n²) shapely)와 전체 재배치가 예산을 초과.

→ **결론: 패킹 밀도보다 "항상 feasible + 시간 준수"가 압도적 우선.**

## 발견 2: 보장된-feasible·시간안전 솔버 (`solver.py`)

베이당 동시 1블록(시간창 비중첩) → 공간충돌·크레인충돌·순서충돌이 **구조적으로
불가능** → 임의 in-bounds 위치에서 항상 feasible. 병렬머신 리스트 스케줄링(EDD,
가장 빨리 끝나는 베이에 배정)으로 tardiness 완화. O(n·m), 사실상 즉시.

`--solver sequential` 전체 40개 결과:

| 지표 | 값 |
|---|---|
| feasible | **40/40 (100%)** |
| 실행시간 | 인스턴스당 ~0.0s (시간초과 불가능) |
| 총 목적값 | 25,488,584,105 |
| 평균 목적값 | 637,214,603 |
| Σ obj1 (tardiness) | 2,232,300 |
| Σ obj2 (불균형) | 286,049 |
| Σ obj3 (선호) | 275,590 |

✅ **−1 위험 제거 → 양수 점수 floor 확보.** 목적값은 **obj1(tardiness)이 지배적**
(w1≈1.3만~2.9만이라 w1·obj1이 목적값 대부분). 직렬화로 블록이 오래 대기하기 때문.

## 발견 3: 공존 그리디 솔버 (`solver.solve_greedy`) — 4.8배 개선

EDD 순서로 블록을 **가장 빨리 끝나는 베이·시각**에 배치하되, 같은 베이에서
공간적으로 양립 가능하면 **공존**시켜 대기(tardiness)를 줄인다.

**Feasible by construction**: `_feasible_insert`로 검증된 배치만 커밋하므로 보수
루프가 없고 절대 infeasible을 내지 않는다. 이 검사는 check_feasibility의 5단계를
모두 모사한다 — 특히 **같은 시각 진입/퇴출**(stage 5)은 경계를 `<=`로 포함해 양방향
크레인 검사를 하는 보수적·순서독립적 방식으로 처리(strict 부등호만 쓰면 stage 5에서
터지는 버그를 수정). 시간예산(0.9·limit) 초과 시 빈-베이 순차 배치로 안전 폴백 →
시간제한 절대 초과 안 함.

`--solver greedy` 전체 40개 결과 (시간제한 30s):

| 지표 | sequential | **greedy** | 개선 |
|---|---:|---:|---:|
| feasible | 40/40 | **40/40** | 유지 |
| 솔버 시간 | ~0.0s | ~28s (<30s 준수) | — |
| 총 목적값 | 25,488,584,105 | **5,348,307,540** | **4.8× ↓** |
| 평균 목적값 | 637,214,603 | **133,707,688** | 4.8× ↓ |
| Σ obj1 (tardiness) | 2,232,300 | **613,548** | 3.6× ↓ |
| Σ obj2 (불균형) | 286,049 | 175,569 | 1.6× ↓ |
| Σ obj3 (선호) | 275,590 | 138,566 | 2.0× ↓ |

인스턴스별 하이라이트: prob_22 322M→1.8M(180×), prob_21 410M→12.7M(32×).

✅ **현재 제출 후보**: `solver.solve_greedy` (myalgorithm 진입점에서 호출). 100%
feasible, 시간안전, baseline(−1) 대비 확실한 양수 점수.
